From setup to live — 5 steps

  1. 1
    Business Profile — add your business: NTN, province, address.
  2. 2
    Add your FBR sandbox token there (from IRIS → Digital Invoicing) and keep Sandbox mode ON.
  3. 3
    Add Buyers and Products — set each product's HS code and sale type once; every invoice reuses them.
  4. 4
    Create Invoice — tax is calculated automatically. Validate, then Submit.
  5. 5
    Once all sandbox scenarios pass, get your production token from IRIS, replace it in Business Profile, and turn Sandbox mode OFF.

If something goes wrong

Invoice stuck on "Pending retry" Network issue — the system retries automatically (5 min → 6 hrs). Check the queue.
"Verify on IRIS portal" message The request may have reached FBR. Check IRIS before resubmitting, to avoid a duplicate invoice.
Rates look outdated Run reference sync, or edit rates (date-effective) in the Admin Panel.

Modes

MOCK
No token configured. Invoices are simulated locally — nothing goes to FBR.
SANDBOX
FBR's test environment. Real API, test records — required for certification.
LIVE
Real invoices with legal effect. Corrections only within 72 hours via IRIS, capped at 10% of last month's sales.
The badge in the top bar always shows the current mode.

Common FBR errors

0002
Buyer NTN must be 7 digits (or CNIC 13)
0044
Item HS code is empty — set it on the product
0046
Rate doesn't match the selected sale type
0062
Steel melting / re-rolling requires UoM "MT"
0090
3rd Schedule goods need a retail price (MRP)
0104
Sales tax must equal value × rate
When validation fails, the invoice screen explains each error inline with a fix.
